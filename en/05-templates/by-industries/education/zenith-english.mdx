<Info>
  **The Business**: Zenith English is a growing English language center managing the complete student lifecycle, from initial marketing campaigns and admissions to academic management and student support.

  **The Challenge**: As Zenith expanded, its operational data became fragmented across different departments. Marketing couldn't track lead quality, Admissions struggled to coordinate with Finance on tuition, and Academic staff lacked a unified view of a student's journey. This created data silos, manual handoffs, and a disconnected experience for both staff and students. This blueprint was designed to solve that by creating a single, unified system for the entire operation.
</Info>

## **OBJECT RELATIONSHIP MODEL**

<Tabs>
  <Tab title="Parent-Child Relationships">
    <Info>
      A built-in, strict hierarchy between a standard `Object` and its `Sub-Objects`. This method is best for work breakdown structures.
    </Info>
    **Application at Zenith English:**

    | Parent Object                      | Child Object(s)                      | Architectural Rationale                                                                                                                                            |
    | :--------------------------------- | :----------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | `Marketing/üßäCampaign`             | `Marketing/üßäÔ∏éAd`                    | **Work Breakdown Structure:** Each `üßäCampaign` is a major plan that is executed through multiple specific `üßäÔ∏éAds`.                                               |
    | `Admissions/üßäLead`                | `Admissions/üßäÔ∏éPlacement Test`       | **Process Breakdown Structure:** A `üßäLead` must complete a `üßäÔ∏éPlacement Test` to determine their proficiency level before enrollment.                            |
    | `Finance/üßäEnrollment Agreement`   | `Finance/üßäÔ∏éInvoice`                 | **Financial Breakdown Structure:** An `üßäEnrollment Agreement`'s total tuition is broken down into multiple `üßäÔ∏éInvoices` for installment payments.                |
    | `Academics/üßäCourse`               | `Academics/üßäÔ∏éClass`                 | **Academic Breakdown Structure:** A `üßäCourse` (e.g., IELTS 7.0) is the curriculum blueprint, which is delivered via multiple `üßäÔ∏éClasses` (e.g., Class I7A, I7B). |
    | `Student Affairs/üßäStudent Record` | `Student Affairs/üßäÔ∏éProgress Report` | **Performance Tracking Structure:** A `üßäStudent Record` contains a chronological series of `üßäÔ∏éProgress Reports` documenting academic performance over time.      |
  </Tab>
  <Tab title="Global Connections">
    <Info>
      A centrally-managed library of reusable relationship _types_ that standardize the _meaning_ of bidirectional relationships across the entire system.
    </Info>
    **Application at Zenith English:** This defines the core student journey between departments.

    | Connection Type                 | Object A                         | A ‚Üí B Relationship | Object B                           | B ‚Üí A Relationship |
    | :------------------------------ | :------------------------------- | :----------------- | :--------------------------------- | :----------------- |
    | **Marketing ‚Üí Admissions**      | `Marketing/üßäAd`                 | Generates          | `Admissions/üßäLead`                | Sourced from       |
    | **Admissions ‚Üî Finance**        | `Admissions/üßäLead`              | Signs              | `Finance/üßäEnrollment Agreement`   | For                |
    | **Finance ‚Üí Student Affairs**   | `Finance/üßäEnrollment Agreement` | Initiates          | `Student Affairs/üßäStudent Record` | Created from       |
    | **Academics ‚Üî Student Affairs** | `Academics/üßäClass`              | Attended by        | `Student Affairs/üßäStudent Record` | Enrolled in        |
  </Tab>
  <Tab title="Object Picker Fields">
    <Info>
      A flexible field that allows one `Object` to reference one or more other `Objects`. This is the tool used to _implement_ the relationships defined above.
    </Info>
    **Application at Zenith English:**

    | Defined on Object                  | Field Name           | Reference Type                            | Purpose                                                                   |
    | :--------------------------------- | :------------------- | :---------------------------------------- | :------------------------------------------------------------------------ |
    | `Admissions/üßäLead`                | `Ad Source`          | Select 1 `Marketing/üßäAd`                 | To track the marketing channel that generated the lead for ROI analysis.  |
    | `Finance/üßäEnrollment Agreement`   | `Student`            | Select 1 `Admissions/üßäLead`              | To specify which lead this agreement is for.                              |
    | `Student Affairs/üßäStudent Record` | `Original Agreement` | Select 1 `Finance/üßäEnrollment Agreement` | To link the student's official record back to their financial agreement.  |
